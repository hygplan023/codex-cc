# Pushing to GitHub using the GitHub connector

**Rule:** The `@replit/connectors-sdk` package is NOT importable inside the `code_execution` sandbox (ERR_MODULE_NOT_FOUND). To get the OAuth token, use the pre-registered `listConnections('github')` instead — the token is at `conns[0].settings.access_token`.

**Why:** The blueprint snippet suggests `new ReplitConnectors()` + `connectors.proxy(...)`, but that SDK only resolves inside the actual artifact project, not the sandbox notebook. `listConnections` works directly in the sandbox.

**How to apply:**
1. `const token = (await listConnections('github'))[0].settings.access_token;`
2. For REST calls: `fetch('https://api.github.com/...', { headers: { Authorization: \`Bearer ${token}\`, 'User-Agent': 'replit-agent', Accept: 'application/vnd.github+json' } })`.
3. For `git push`: run via Node `child_process.execFile` inside the sandbox with a one-time URL `https://x-access-token:${token}@github.com/<owner>/<repo>.git` and push `main:main`. Do NOT `git remote add` with the token (avoids persisting it). Always redact the token from any stdout/stderr before logging.
4. `git push` (no force) and `git remote add` are non-destructive and runnable directly; `git commit`/`git rm`/force-push must be delegated to a background Project Task.

**Gotcha:** `dist-package.zip` (generated by the download endpoint) can end up tracked in git and bloats the repo. Add it to `.gitignore` and keep it untracked.
